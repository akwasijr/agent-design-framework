# Getting Started

## The Core Insight

> **Work is shifting from doing to managing. Every knowledge worker becomes a manager: not of people, but of agents.**

Today, a loan officer processes applications. A doctor documents encounters. A government clerk reviews permits. Tomorrow, agents handle the processing, documentation, and review. The human sets parameters, reviews edge cases, exercises judgment, and provides strategic direction.

If every worker becomes a manager, **every product becomes a management tool**. The design vocabulary must shift: from task completion to oversight interfaces, from data entry to decision-making, from process flows to exception handling, from settings panels to behavioral boundaries.

| Role Today | Role Tomorrow | Core Shift |
|---|---|---|
| Processor | Governor | Doing to Overseeing |
| Assessor | Adjudicator | Evaluating to Judging |
| Creator | Creative Director | Making to Directing |
| Practitioner | Care Architect | Treating to Orchestrating |
| Administrator | Public Steward | Processing to Governing |

## Why This Framework

Agent technology is advancing rapidly, but **design thinking hasn't kept up**. Three layers are missing:

1. **The Experience Layer.** No design language exists for agent-managed work. We need new patterns for expressing intent, reviewing agent work, and exercising judgment.
2. **The Accessibility Layer.** Agent creation is currently a technical skill. If agents are the future of all work, creating them must be as accessible as making a spreadsheet. See [Workshop Toolkit](/workshop-toolkit) for methods to bridge this gap.
3. **The Governance Layer.** No standard exists for defining agent boundaries, auditing decisions, or assigning responsibility. In banking, healthcare, and government, this is essential.

Once the first generation of agent tools ships, patterns harden and become difficult to change. This framework helps designers, product leaders, and technologists think clearly about this transition before patterns are set.
